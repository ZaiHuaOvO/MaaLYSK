> 完整 MaaLYSK 软件压缩包命名格式为：MaaLYSK-平台-架构-版本.zip
>
> 例如Windows用户在绝大部分情况下，都需要使用x64架构的 MaaLYSK ，即需要下载 MaaLYSK-win-x64-vx.x.x.zip

### 安装包版本选择

- **Windows**

  对于绝大部分用户，下载  `MaaLYSK-win-x86_64-vx.x.x.zip` 。

  若确定自己的电脑是 arm 架构，下载  `MaaLYSK-win-aarch64-vx.x.x.zip`（暂未测试）

- **Mac**

  M 芯片用户，下载  `MaaLYSK-macos-aarch64-vx.x.x.zip`（暂未测试）

  Intel 芯片用户，下载 `MaaLYSK-macos-x86_64-vx.x.x.zip`

  > [!IMPORTANT]
  >
  > 因mac开发、打包和测试资料过少，以及未知的原因
  >
  > Mac用户请都按照`Intel 芯片`进行之后的步骤，无论是下载还是环境安装都以`Intel 芯片`进行

- **Linux**

  x64架构，下载 `MaaLYSK-linux-x86_64-v0.1.4.zip`（暂未测试）

  否则下载`MaaLYSK-macos-aarch64-vx.x.x.zip`（暂未测试）

## Windows

推荐使用[MuMu模拟器12](https://mumu.163.com/)，后续步骤均会以该模拟器进行讲解。

### 模拟器设置

1. 新建一个设备（随便设置）后启动模拟器，选择右上角的三个点，选择`设备设置`

2. 在`性能`-`性能设置`中，处理器设置为`2核及以上`，运行内存设置为`2GB及以上`

   ![模拟器性能设置](/image/tutorial_image/win/模拟器设置0.png)

   > [!NOTE]
   >
   > 性能通常推荐设置得越高越好~~（因为恋与深空真的很卡）~~
   >
   > 如果模拟器出现明显的卡顿、卡死现象，请及时提升性能，可能会影响到Maa的识别和操作。
   >
   > 深空卡死还有可能是画质渲染模式问题，尝试切换

3. 在`显示`-`分辨设置`中，选择手机，并选择分辨率`900*1600`-`DPI320`-`手机`

   ![模拟器分辨率设置](/image/tutorial_image/win/模拟器设置1.png)

4. 在`其他`-`应用保活`中，确认`后台挂机时保活运行`处于**未勾选**（关闭）状态

   ![模拟器应用保活](/image/tutorial_image/win/模拟器设置2.png)

5. 点击设置界面右上角的`×`后，选择`立即重启`

   <img src="/image/tutorial_image/win/模拟器设置3.png" alt="模拟器重启" width="70%"/>

### 运行库安装

好消息：自v0.3.0版本之后，windows支持点击MaaLYSK.exe后，自动安装运行库了！

如果自动安装失败，尝试下方步骤进行运行库安装！

> 接下来若系统出现任何安全性的弹窗提醒，请均点击`允许`/`同意`
>
> 有的弹窗比较隐蔽，则找一找有没有`下拉三角符号`/`更多`之类的提示，并选择`仍然保留`/`仍然运行`

- 右键项目目录下的`DependencySetup_ 依赖库安装_ win.bat文件`，选择`以管理员身份运行`进行运行库安装，等待安装完成后并重启计算机。安装成功的界面参考如下：

  ![环境准备](/image/tutorial_image/win/环境准备-1.png)

::: details 若安装失败（出现许多红色报错）

- 上述方法安装失败，可点击下方运行库的官网链接跳转并逐个安装。

  - [`Visual C++ 可再发行程序包`](https://aka.ms/vs/17/release/vc_redist.x64.exe)

  - [`.NET 桌面运行时 10（.NET Desktop Runtime 10.0.0）`](https://dotnet.microsoft.com/en-us/download/dotnet/10.0#:~:text=Binaries-,Windows,-x64)

    > x64 架构，下载`.NET 10 Desktop Runtime for Windows x64`（如果你不知道选择哪个，就下载这个）
    >
    > arm 处理器，下载`.NET Desktop Runtime 10 for Windows Arm64`

    ![环境准备](/image/tutorial_image/win/环境准备0.png)

    > [!NOTE]
    >
    > 请确保你所下载的是`.NET Desktop Runtime x.x.x`，不是`SDK`、不是`ASP.NET Core Runtime`

:::

::: details 运行库官网下载缓慢或失败

前往QQ群-群文件-MaaLYSK运行库（选择对应版本安装），下载运行库

> window都需下载`VC_redist.x64.exe`
>
> x64 架构，下载`windowsdesktop-runtime-10.0.0-win-x64.exe`
>
> arm 处理器，下载`windowsdesktop-runtime-10.0.5-win-arm64.exe`

> [!CAUTION]
>
> 如果不能右键`DependencySetup_ 依赖库安装_ win.bat文件`安装的话，其它方式点击**下载**后，只是下好了运行库的**安装包**，而非**完成**了运行库的安装。
>
> 需要**以管理员身份运行**两个exe文件完成安装！

:::

> [!WARNING]

### 连接设置

#### 自动连接

当你第一次启动MaaLYSK时，它会自动识别并查找你的模拟器地址

![连接设置](/image/tutorial_image/win/连接设置.png)

- 如果出现了意外（连接失败、找不到模拟器），可以尝试点击`刷新`按钮（右上角的🔄按钮）

- 多开模拟器，则选择`当前控制器`的`下拉`按钮，选择对应的模拟器`端口号`。（端口号相关说明参照[常见问题 |连接问题-手动连接模拟器](https://maalysk.top/zh_cn/manual/2.2.html#手动连接模拟器)）

通常默认的安装地址：`C:/Program Files/Netease/MuMuPlayer-12.0`

#### 手动连接

MaaLYSK自动连接失败？参照[常见问题 |连接问题-手动连接模拟器](https://maalysk.top/zh_cn/manual/2.2.html#手动连接模拟器)

## MacOS

M 芯片推荐[蓝叠模拟器 Air 版](https://www.bluestacks.com/mac)，后续会只按照该模拟器进行讲解。

> 本项目不支持 PlayerCover，而MuMu Pro模拟器要钱~~（把 mac 用户当韭菜呢）~~

Intel 芯片及其他模拟器暂无教程提供，如需要可参考 MAA 官方教程：[Mac 模拟器｜MAA 文档站](https://docs.maa.plus/zh-cn/manual/device/macos.html)

> Intel 芯片推荐使用 Mac 自带的多系统安装 Windows ，并使用 Windows 版

### 模拟器设置

1. 选择右上角的三根横线，选择`设置`

   ![模拟器横线](/image/tutorial_image/mac/模拟器横线.png)

2. 设置中文

   ![模拟器设置中文](/image/tutorial_image/mac/模拟器设置中文.png)

3. 设置 ADB

   ![模拟器设置ADB](/image/tutorial_image/mac/模拟器设置ADB.png)

4. 在性能中，设置为`高性能`（可选）

5. 设置分辨率为`900*1600`-`竖屏`后重启

   ![模拟器设置分辨率](/image/tutorial_image/mac/模拟器设置分辨率.png)

### 运行库安装

- [下载链接](https://dotnet.microsoft.com/en-us/download/dotnet/10.0)，下载后直接运行安装

  > Intel 处理器，下载 `.NET 10 for macOS x64`
  >
  > M1, M2 等 arm 处理器，下载 `.NET 10 for macOS Arm64`

  ![环境安装0](/image/tutorial_image/mac/环境安装0.png)

::: details 运行库官网下载缓慢或失败：

  前往QQ群-群文件-MaaLYSK运行库（选择对应版本安装），下载mac的运行库。

  >  Intel 处理器，下载`dotnet-runtime-10.0.5-osx-x64.pkg`
  >
  >  M1, M2 等 arm 处理器，下载 `dotnet-runtime-10.0.0-osx-arm64.pkg`

### 启动与权限设置

1. 将下载好的MaaLYSK**解压缩**之后，右击文件夹，选择`服务`-`新建位于文件夹位置的终端窗口`

   > 如果没有 Mac 解压软件推荐 `Keka`，App Store 即可下载。

   ![环境安装1](/image/tutorial_image/mac/环境安装1.png)

2. 在终端中依次输入指令

   ```shell
   chmod a+x MFAAvalonia
   ./MFAAvalonia
   ```

3. 输入完指令之后会弹出窗口并提示：因为Apple无法检查其是否包含恶意软件……

   ![环境安装2](/image/tutorial_image/mac/环境安装2.png)

4. 此时，请前往 `设置` - `隐私与安全性`，下滑至页面底部，找到“已阻止使用的应用”，点击 `仍要打开`/`仍然允许`。请设置窗口保持打开，由于涉及多个文件，需要依次进行允许。

   ![环境安装3](/image/tutorial_image/mac/环境安装3.png)

   > 只要隐私与安全性中出现 `仍要打开`/`始终允许`，优先点击，再选择弹窗中的选项 `打开`/`在XX中显示/打开`（不要选择`移到废纸篓`/`取消`）

5. 再次在终端输入`./MFAAvalonia`

6. 如果没有安装运行库或运行库安装有问题，就会出现如图这样的下载链接，复制链接到浏览器进行下载安装运行库，重新执行`./MFAAvalonia`即可

   ![环境安装4](/image/tutorial_image/mac/环境安装4.png)

### 连接设置

#### 自动连接

1. 设置快捷方式，在项目文件夹中找到`MFAAvalonia`或`MaaLYSK`，右键选择`制作替身`，生成的新文件就是本项目的快捷方式了，双击即可打开`MaaLYSK`（放在程序坞、快捷栏均可）

   <img src="/image/tutorial_image/mac/软件设置0.png" alt="软件设置0" width="60%" />

2. 如果你已经安装了模拟器，当你第一次启动MaaLYSK时，它会自动识别并查找你的模拟器地址，如图所示（5554是蓝叠模拟器的默认端口）。

   > 如果出现了意外（连接失败、找不到模拟器），可以尝试点击`刷新`按钮（右上角的🔄按钮），或手动选择应用程序中的`BlueStacks`

   ![软件设置1](/image/tutorial_image/mac/软件设置1.png)

   - 多开模拟器，则选择`当前控制器`的`下拉`按钮，选择对应的模拟器`端口号`。（端口号相关说明参照[常见问题 |连接问题-手动连接模拟器](https://maalysk.top/zh_cn/manual/2.2.html#手动连接模拟器)）

#### 手动连接

参照[常见问题 |连接问题-手动连接模拟器](https://maalysk.top/zh_cn/manual/2.2.html#手动连接模拟器)
