# 文档编写

## 获取源码

  项目地址：[MaaLYSK](https://github.com/Witty36/MaaLYSK)

  将项目Fork并Clone保存到本地后，使用编辑器打开（推荐`VsCode`）

### 项目结构

  文档站的核心目录是`/docs`，开发和更新都只需要在这个目录中。

```js
  docs
  ├─ .vitepress              # VitePress 配置
  │  ├─ cache
  │  ├─ dist
  │  └─ config.mts           # 配置文件
  │
  ├─ en_us                   # 英文文档
  │  ├─ develop              # 英文-开发文档
  │  ├─ manual               # 英文-使用手册
  │  └─ index.md             # 英文首页
  │
  ├─ zh_cn                   # 中文文档
  │  ├─ develop              # 中文-开发文档
  │  ├─ manual               # 中文-使用手册
  │  └─ index.md             # 中文首页
  │
  ├─ public                  # 静态资源
  │  ├─ doc_image            # 文档图片
  │  └─ logo.ico             # 网站图标
  │
  ├─ .markdownlint.yaml
  └─ index.md                # 根首页（入口）

```

## 环境安装

### 安装Node.JS

  点击前往下载=>[Node.JS](https://nodejs.org/zh-cn/download)，版本请确保**>18**。

> 如何确认Node.JS安装成功：
>
> 打开终端（Windows用户可按`Win`+`R`，输入`cmd`后回车）
>
> 在终端中输入`node -v`后回车，出现版本号即代表安装成功

### 安装VitePress

  在 MaaLYSK 项目根目录下终端执行（以npm为例）

```shell
npm add -D vitepress@next
```

> 如果你不知道如何打开项目根目录的终端，可以参考以下方式：
>
> - VsCode界面按下`Ctrl`+`Shift`+`~` ，即可打开VsCode终端
> - （仅Windows可用）在MaaLYSK本地文件夹内部，在Windows资源管理器上方的地址栏输入`cmd`后回车

### 本地运行

  在终端中执行

```shell
npm run docs:dev
```

### 打包测试

  如果你想确认你的项目是否能成功编译并渲染，可以在终端中执行

```shell
npx vitepress dev docs
```

  更详细的相关文档可访问：[VitePress：快速开始](https://vitepress.dev/zh/guide/getting-started)

## 文档规范

### Markdown语法标准

请确保所有文档的内容都符合Markdown官方标准语法标准，如果还不了解可以前往进行[Markdown语法教程](https://markdown.com.cn/)学习（很简单的）

### 图片引用

文档所需要的所有图片，请确保存放地址为`MaaLYSK/assets/img`目录下，并上传至你的Github仓库，该目录下你可以自行创建子目录

假设有一张图片的绝对路径为`MaaLYSK/assets/img/图片名.png`，在文档中你需要引用直链：

```markdown
![图片名](https://raw.githubusercontent.com/你的用户名/MaaLYSK/main/assets/img//图片名.png)
```

假设有一张图片的绝对路径为`MaaLYSK/docs/public/docs_image/图片名.png`，在文档中引用的用法为

```markdown
![图片名](https://raw.githubusercontent.com/你的用户名/MaaLYSK/main/assets/img/docs_image/图片名.png.png)
```

如果你对于图片有一定的样式要求（例如调整宽度），你可以使用HTML语法去实现，例如：

```html
<img src="https://raw.githubusercontent.com/你的用户名/MaaLYSK/main/assets/img/图片名.png" alt="图片名" style="width:50%" />
```

在提交PR时，请确保所有图片引用都是指向父仓库`Witty36/MaaLYSK`，例如：

```
![图片名](https://raw.githubusercontent.com/Witty36/MaaLYSK/main/assets/img/docs_image/图片名.png.png)
```

### 命名格式

  所有文档的命名方式为：`x.x-文档名`

  其中 `x.x` 在它的目录下必须是唯一的，不允许重复。

> 例如manual和develop都可以有一篇1.1，但manual不可以有两篇1.1

  `x.x` 拥有排序功能。

> 例如文档的顺序是：0.0，0.1，1.1，1.2，2.1……

  `x.x` 中的两个x分别对应`父模块`和`子模块`

> 例如manual中的1.x，对应的是第一章新手入门/快速开始相关的文档
>
> 例如develop中的2.x，对应的是文档站开发相关文档

### 语法格式

- 文档应当有且仅有一个一号标题，为本文档的主标题。

- 主标题的图标可访问[REMIX ICON](https://remixicon.com/)，复制对应的i标签代码例如

  ```html
  <i class="ri-rocket-line"></i>
  ```

- 文档的标题请勿带`x.x`，例如文件名叫`1.1-快速开始.md`的文档，标题应该为一号标题的`快速开始`

- 只有二号标题会识别为快捷目录，其余标题均不会被识别。

- 文档中若是使用了HTML代码，请在文档站本地测试确认样式无误后再提交，Markdown编辑器和文档站的预览会有所差异，以文档站的显示为准。
